## Purpose

The repository has strong generated-domain evidence, but the largest scientific
gap remains operator realism. If the forward model is wrong, inversion can
explain solver or sensor mismatch as via, return path, model gap, or layer
misallocation.

E21 should build a small auditable ladder that compares forward operators on
the same current geometry and reports how current-inversion conclusions change
when the operator changes.

This is a breakthrough direction because it attacks inverse crime directly.

## First-Principles Formulation

Let:

```text
y_k = A_k x + epsilon_k
```

where `A_k` is one of:

- analytic canonical operator;
- centerline Biot-Savart;
- finite-width surrogate;
- PyPEEC;
- optional external solver artifact, if available.

The operator gap between `A_i` and `A_j` on a current family `X` is:

```text
Gap(i, j; X) =
E_x || W (A_i x - A_j x) ||_2 / || W A_j x ||_2
```

But field gap alone is insufficient. The key scientific quantity is conclusion
instability:

```text
DecisionGap(i, j) =
Pr[ decision_i(y_j) != decision_j(y_j) ]
```

where `decision_i` is an inverse or hypothesis scorer calibrated or built under
operator `A_i`, and `y_j` is generated by operator `A_j`.

E21 should report both field-level and decision-level operator gap.

## Solver Ladder

The implementation should support a ladder:

1. analytic canonical wire/loop/via sanity;
2. centerline Biot-Savart;
3. finite-width surrogate;
4. PyPEEC where available;
5. optional external solver artifact ingestion:
   - COMSOL CSV/NPZ placeholder interface;
   - FastHenry/Fasthenry-like exported currents/fields placeholder interface;
   - no claim if actual files are absent.

The first runnable slice must produce evidence even if COMSOL/FastHenry are
absent, but it must clearly report that external validation is missing.

## Algorithm Steps

1. Define a small canonical geometry set: straight wire, loop, finite-width trace,
   via/vertical segment, return path pair, small multilayer route.
2. Build or adapt forward outputs for each available operator.
3. Normalize coordinate frame, units, current scale, sensor plane, and component
   order.
4. Compute operator gap metrics: component RMSE, relative field error, spectral
   error by spatial frequency, sign/polarity consistency, divB proxy residual.
5. Run a simple inverse or hypothesis scorer under one operator and evaluate on
   another: same-operator, centerline -> finite-width, centerline -> PyPEEC,
   finite-width -> PyPEEC.
6. Report decision instability and failure mechanisms.
7. If external solver artifacts are absent, write a valid blocked/interface
   result, not a fake validation.

## Metrics and Gates

Required metrics: case_count, available_operator_count,
external_solver_artifacts_present, centerline_vs_finite_width_field_rel_rmse,
centerline_vs_pypeec_field_rel_rmse, finite_width_vs_pypeec_field_rel_rmse,
spectral_low_k_gap, spectral_high_k_gap,
decision_instability_centerline_to_finite_width,
decision_instability_centerline_to_pypeec, polarity_consistency_rate,
unit_sanity_passed, external_validation_status.

Engineering gates: package runs in smoke; all available operators return same
shape and units; external artifact loader has schema tests; missing external
artifacts result in blocked/interface status, not crash; reports are written;
generated-domain and external-validation boundaries are explicit.

Scientific gates: field-level operator gaps are nonzero and quantified; at least
one decision-level stress is reported; if external artifacts are absent, claim
remains scaffold/blocked; if PyPEEC is used, results are explicitly marked
generated-domain.

## Cannot Claim

E21 cannot claim:

- COMSOL/FastHenry/FEM validation unless real external artifacts are loaded;
- PyPEEC is ground-truth real physics;
- generated operator agreement proves real CAD/GDS or real QDM/NV validation;
- inverse decisions transfer to real hardware.

E21 can claim only:

- quantified operator gaps among implemented/generated operators;
- an external artifact interface if no external artifacts are present;
- decision instability under solver swaps in the tested domain.
